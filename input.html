File Input

Purpose:

This part of the program will scan through the configuration file, determining the initial region file, time limit, and refresh rate of the simulation loop. The part of the program will also read through the region file, which is a csv format file, storing it for simulation.

Data Storage & Maintenance:

The datastructures used are basic structs and Vectors in C++. The datastructures used here are Configuration, RegionState, and Zone. Essentially, the processConfigFile() functions reads in the given .txt file that contains the following lines in order and exact format:

  1. Region Layout:region1.csv
  2. Time Limit:20
  3. Refresh Rate:1

The data gathered from this .txt file is then stored in a Configuration variable for easier use. This variable is given to the processConfigFile() function as a reference to be able to use globally.

After the main config file is read, the region file is read based on the name gathered from the config file. This is taken care with the processRegionFile() function that takes in the region file name and the RegionState variable. The RegionState variable is passed in as a reference to be able to use globally.

The final step of the File Input Functionality is the output of the initial state and the update loop which the simulation will run in. The output of the simulation state is taken care of by the outputRegionState() function. The simulation loop runs in the main.cpp and runs until the maxTimeSteps value in the Configuration structure is met.

Functionality:

This compenent is aminly setting up the initial data of the simulation and so has mostly been covered in the "Data Storage & Maintenance" section. But lets talk about the flow of the functions involved in this functionality.

The 2 functions involved in this functionality are processConfigFile() and processRegionFile(). The first fucntion receives the data from the main configuration file and stores it in a Configuration variable. The second function then uses the name of the initial region .csv file found in the Configuration variable to read and store the city in a RegionState variable.
