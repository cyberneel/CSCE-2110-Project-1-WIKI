Description:

Our CSCE 2110 Project 1 that simulates the growth of a city over time and was written in C++. The simulation is broken down into 5 "Major Functionalities":

  1. File Input
  2. Commercial Update
  3. Industrial Update
  4. Residential Update
  5. Pollution Update

these 5 components make up our simulation and run in that order.

Before we get into what exactly each component does, let's look at the code graph that represents the entire system to get a better idea of how they are interconnected!

At a high level, Commercial activities are prioritized over others. This is why CommercialUpdate() and IndustrialUpdate() take place before the ResidentialUpdate() and PollutionUpdate().
The ResidentialUpdate() provides the city with workers. Similarly, the IndustrialUpdate() provides goods for the CommercialUpdate(). These functions will be further explained later.

Now lets look at each Component Seperately.

  • File Input The reading of the initial state and configurations for the simulation
  • Commercial Update The function that takes care of the commercial Zone s' changes
  • Industrial Update The function that takes care of the industrial Zone s' changes
  • Residential Update The function that takes care of the residential Zone s' changes
  • Pollution Update The function that applies the pollution rules to the Zone s
  • Datastructures
    • Configuration keeps track of the settings for the simulation
    • Zone holds the attributes we need to know for each Zone
    • RegionState keeps track of the data relevent to the entire city
